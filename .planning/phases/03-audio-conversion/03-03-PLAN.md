**Create `OhMyClaw/Audio/CSVWriter.swift`:**

CSVRow struct (Sendable):
- Properties: filename (String), title (String), artist (String), album (String), format (String), bitrate (String), date (String)
- Computed `var csvLine: String` — joins all fields comma-separated, each passed through `escapeCSV(_:)`
- Private `escapeCSV(_ value: String) -> String` — if value contains comma, double-quote, or newline: wrap in double-quotes and escape internal quotes by doubling (`"` → `""`) per RFC 4180. Else return as-is.

CSVWriter struct (Sendable):
- `let fileURL: URL`
- Private static let `headers = "Filename,Title,Artist,Album,Format,Bitrate,Date"`
- `func append(row: CSVRow) throws`:
  - If file doesn't exist: create directory (withIntermediateDirectories), write headers + newline
  - Open FileHandle(forWritingTo:), seekToEndOfFile(), write row.csvLine + newline, close handle
- Date format: ISO 8601 date-only (`yyyy-MM-dd`) — use DateFormatter or ISO8601DateFormatter with date-only style

**Modify `OhMyClaw/Audio/AudioTask.swift`:**

Add new dependencies to AudioTask init:
- `ffmpegPath: URL?` — nil when ffmpeg not installed (degraded mode)
- `conversionPool: ConversionPool?` — nil when ffmpeg not installed
- `qualityCutoff: QualityTier` — from config
- `csvWriter: CSVWriter` — for low-quality logging

After the existing duplicate check (step 4), replace the current move-to-~/Music logic (step 5) with quality evaluation and three branches:

**Step 5: Evaluate quality**
- Call `QualityEvaluator.resolveTier(format: metadata.format, bitrateKbps: metadata.bitrateKbps)`
- Call `QualityEvaluator.isHighQuality(tier: tier, cutoff: qualityCutoff)`

**Step 6a: High quality + AIFF source format → move directly to ~/Music**
- Per user decision: AIFF is already target format, skip conversion entirely
- Move file to ~/Music (reuse existing move logic with filename conflict handling)
- Update library index
- Return `.processed(action: "Moved AIFF to ~/Music (no conversion needed)")`

**Step 6b: High quality + non-AIFF + ffmpeg available → convert + move**
- Per user decision: WAV source files also get converted to AIFF for consistency
- Acquire ConversionPool slot: `await conversionPool!.acquire()`
- Build destination: musicDir/filename-with-.aiff-extension (replace original extension with .aiff)
- Call `try await FFmpegConverter.convert(input: file, output: destination, ffmpegPath: ffmpegPath!)`
- Release pool slot — use explicit do/catch (not defer+Task, per research recommendation):
  ```
  do {
      try await FFmpegConverter.convert(...)
      await conversionPool!.release()
  } catch {
      await conversionPool!.release()
      // Conversion failure: leave original untouched per user decision
      AppLogger.shared.error("Conversion failed", context: [...])
      return .error(description: "ffmpeg conversion failed: \(error)")
  }
  ```
- Delete original from ~/Downloads: `try FileManager.default.removeItem(at: file)`
- Update library index with converted file's path
- Return `.processed(action: "Converted to AIFF and moved to ~/Music")`

**Step 6c: High quality + non-AIFF + NO ffmpeg → degraded mode**
- Per user decision: without ffmpeg, skip conversion, move original to ~/Music
- Move file to ~/Music in original format (reuse move logic)
- Update library index
- Log warning about degraded mode
- Return `.processed(action: "Moved to ~/Music (ffmpeg unavailable, original format)")`

**Step 6d: Low quality or unknown format → quarantine**
- Build low_quality dir: `musicDir.appendingPathComponent("low_quality", isDirectory: true)`
- Create directory if needed
- Check if same filename already exists in low_quality: if yes, skip the move (don't overwrite, per user decision)
- If not duplicate: move file from ~/Downloads to ~/Music/low_quality
- Log to CSV via csvWriter.append with CSVRow(filename, title, artist, album, format extension, bitrate string, today's date)
- Delete original from ~/Downloads (per user decision: same cleanup as high-quality path)
- Return `.processed(action: "Quarantined to ~/Music/low_quality")`

**Consolidate the move logic** into a private helper method to reduce duplication:
- `private func moveFile(_ file: URL, to destination: URL) throws -> URL` — handles filename conflicts via possible_duplicate/ subdirectory (existing pattern), returns final destination URL

Key behavior from user decisions:
- Pipeline order: duplicate check runs BEFORE quality eval (avoid wasting CPU)
- Conversion failure: leave original file untouched in ~/Downloads
- Original cleanup: delete original AFTER successful conversion+move or quarantine+move
- Low-quality duplicate: if same filename exists in ~/Music/low_quality, skip move (no overwrite, no suffix)
