  <name>Task 1: FFmpegLocator & FFmpegConverter</name>
  <files>OhMyClaw/Audio/FFmpegService.swift</files>
  <action>
Create `OhMyClaw/Audio/FFmpegService.swift` with three components:

**ConversionError enum** (Sendable):
- `ffmpegFailed(exitCode: Int32, stderr: String)` — non-zero ffmpeg exit
- `processLaunchFailed(Error)` — Process.run() threw

**FFmpegLocator struct** (Sendable):
- `static func locate() -> URL?`
- Search order: `/opt/homebrew/bin/ffmpeg` (Apple Silicon), `/usr/local/bin/ffmpeg` (Intel), then PATH via `/usr/bin/which`
- For direct paths: use `FileManager.default.isExecutableFile(atPath:)`
- For PATH fallback: create `Process` with `executableURL: URL(fileURLWithPath: "/usr/bin/which")`, `arguments: ["ffmpeg"]`, capture stdout via `Pipe`, call `run()` + `waitUntilExit()`, parse path from output
- Return `URL(fileURLWithPath:)` of first found executable, or nil

**FFmpegConverter struct** (Sendable):
- `static func convert(input: URL, output: URL, ffmpegPath: URL) async throws`
- Build temp output path: `output.deletingPathExtension().appendingPathExtension("aiff.tmp")` — use UUID prefix in temp directory instead: `FileManager.default.temporaryDirectory.appendingPathComponent(UUID().uuidString + ".aiff")` to avoid race conditions between files with the same name
- Create `Process` with `executableURL: ffmpegPath`
- Arguments: `["-i", input.path, "-f", "aiff", "-acodec", "pcm_s16be", "-y", tempOutput.path]`
- Attach `Pipe()` to both `standardOutput` and `standardError`
- Use `withCheckedThrowingContinuation` wrapping `process.terminationHandler`:
  - On success (`terminationStatus == 0`): atomic rename — if output exists, `removeItem`, then `moveItem(at: tempOutput, to: output)`. Resume continuation.
  - On failure: delete tempOutput with `try?`, read stderr from pipe (`readDataToEndOfFile()` after termination — safe for audio-only conversion), resume with `ConversionError.ffmpegFailed`
- Wrap `process.run()` in do/catch — on failure, clean up temp, resume with `ConversionError.processLaunchFailed`

Key constraints per user decisions:
- Write to temp file first, atomic rename on success — never leave corrupt .aiff in destination
- Delete temp on any failure path
- No `-ar` or `-ac` flags — preserve source sample rate and channels

<task type="auto">
  <name>Task 2: ConversionPool actor</name>
  <files>OhMyClaw/Audio/ConversionPool.swift</files>
  <action>
Create `OhMyClaw/Audio/ConversionPool.swift`:

**ConversionPool actor**:
- `private let maxConcurrent: Int`
- `private var inFlight: Int = 0`
- `private var waiters: [CheckedContinuation<Void, Never>] = []`
- `init(maxConcurrent: Int = ProcessInfo.processInfo.processorCount)` — default to CPU core count per AUD-09
- `func acquire() async`:
  - If `inFlight < maxConcurrent`: increment `inFlight`, return immediately
  - Else: suspend via `await withCheckedContinuation { continuation in waiters.append(continuation) }`, then increment `inFlight`
- `func release()`:
  - Decrement `inFlight`
  - If `!waiters.isEmpty`: `let waiter = waiters.removeFirst(); waiter.resume()`

This actor-based semaphore pattern matches the project's existing actor usage (MusicLibraryIndex). The acquire/release API will be called by AudioTask in Plan 03-03 around each FFmpegConverter.convert call.
