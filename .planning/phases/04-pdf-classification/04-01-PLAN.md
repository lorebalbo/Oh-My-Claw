  <action>
Create `OhMyClaw/PDF/PDFTextExtractor.swift`:

**Imports:** Foundation, PDFKit

**PDFMetadata struct** (Sendable):
- `let title: String?`
- `let author: String?`
- `let subject: String?`

**PDFTextExtractor struct** (Sendable):
- Private constants: `maxWords = 1500`, `fallbackPageCount = 3`
- Main method: `func extract(from url: URL) -> (text: String, metadata: PDFMetadata)?`

**extract(from:) implementation:**

All PDFKit work must happen synchronously in one block (PDFKit is NOT thread-safe per research pitfall #4). The return type provides Sendable String + PDFMetadata for async consumption.

1. Guard: `PDFDocument(url: url)` — return nil if init fails (corrupt/unreadable PDF)
2. Guard: `!document.isLocked` — return nil if password-protected (user decision: skip, leave in Downloads)
3. Extract metadata from `document.documentAttributes`:
   - `PDFDocumentAttribute.titleAttribute` as? String
   - `PDFDocumentAttribute.authorAttribute` as? String
   - `PDFDocumentAttribute.subjectAttribute` as? String
   - Build `PDFMetadata(title:, author:, subject:)`
4. Try abstract detection first:
   - Get first page text: `document.page(at: 0)?.string ?? ""`
   - Call `extractAbstract(from: firstPageText)` (private method, see below)
   - If abstract found: clean up, cap words, return (text, metadata) — but only if non-empty after cleanup
5. Fallback: extract first N pages:
   - `let limit = min(document.pageCount, fallbackPageCount)`
   - Loop 0..<limit: get `page.string`, skip nil/whitespace-only strings
   - Join with "\n"
6. Clean up extracted text, cap at maxWords
7. Return nil if final text is empty (image-only PDF — no text layer on any page)

**extractAbstract(from:) private method:**
- Split text by newlines
- Scan for abstract header: case-insensitive check against markers: `["abstract", "summary", "résumé", "zusammenfassung", "riassunto"]`
  - Use `trimmed.hasPrefix($0)` for each marker
- Once abstract start found, scan for section-end markers: `["introduction", "1.", "1 ", "keywords", "key words", "i.", "i ", "background"]`
  - Use `trimmed.hasPrefix($0)` for each marker
- If start found but no end marker: cap at ~30 lines after start
- Join abstract lines with " ", trim whitespace
- Return nil if result is < 50 characters (too short to be a real abstract)

**cleanup(_:) private method:**
- Collapse multiple whitespace/newlines into single spaces: regex `\\s+` → `" "`
- Strip standalone page number patterns: regex `\\b\\d{1,4}\\b(?=\\s|$)` → `""`
- Trim leading/trailing whitespace

**capWords(_:max:) private method:**
- Split by whitespace, filter empty strings
- If word count ≤ max: return original text
- Otherwise: take first `max` words, rejoin with " "

Key constraints from user decisions:
- Token cap: ~2000 tokens ≈ ~1500 words — use simple word count, not tokenizer
- Abstract detection: heuristic keyword matching — acceptable if imperfect
- Language-agnostic: abstract markers include non-English terms
- PDFKit thread safety: all work synchronous, return Sendable types
