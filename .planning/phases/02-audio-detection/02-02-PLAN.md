  <action>
Create `OhMyClaw/Audio/MusicLibraryIndex.swift` with a Swift actor for thread-safe duplicate detection.

**MusicLibraryIndex actor:**

**Properties:**
- `private var index: [String: URL] = [:]` — keyed by normalized "title|artist" string, value is file URL
- Private references to `AudioFileIdentifier` and `AudioMetadataReader` (injected or created internally)

**Key normalization (private):**
- `private static func normalizeKey(title: String, artist: String) -> String`
- Trim whitespace+newlines from both, lowercase both
- Return `"\(title)|\(artist)"` (per user decision: exact match after trim+lowercase)

**Build method:**
- `func build(from musicDirectory: URL) async`
- Use `FileManager.default.enumerator(at:includingPropertiesForKeys:[.isRegularFileKey], options:[.skipsHiddenFiles])` to recursively enumerate ~/Music
- Collect all URLs where `AudioFileIdentifier.isRecognizedAudioFile` returns true
- Use `withTaskGroup(of: (String, URL)?.self)` with bounded concurrency (max 8 parallel reads) to read metadata via AudioMetadataReader
- For each file with non-nil title AND artist, add `normalizeKey(title:artist:) → url` to index
- Skip files where metadata read fails (try? — don't crash on unreadable files)
- Log total indexed count and elapsed time via AppLogger at INFO level: "Music library indexed", context: ["count": "\(index.count)", "elapsed": "Xs"]

**Query methods:**
- `func contains(title: String, artist: String) -> Bool` — normalize key, check index
- `func url(for title: String, artist: String) -> URL?` — normalize key, return stored URL

**Mutation methods:**
- `func add(title: String, artist: String, url: URL)` — normalize key, insert into index
- `func remove(title: String, artist: String)` — normalize key, remove from index

**Bounded concurrency pattern for build:** Submit up to 8 tasks before awaiting results. When limit reached, await one result before submitting next. Collect remaining results after loop. Per research: prevents file descriptor exhaustion on large libraries.

<task type="auto">
  <name>Task 2: Create AudioTask conforming to FileTask</name>
  <files>OhMyClaw/Audio/AudioTask.swift</files>
  <action>
Create `OhMyClaw/Audio/AudioTask.swift` implementing the `FileTask` protocol as the full audio pipeline orchestrator.

**AudioTask struct conforming to FileTask:**

**Properties:**
- `let id = "audio"`
- `let displayName = "Audio Detection"`
- `let isEnabled: Bool` (from config.audio.enabled)
- `private let identifier: AudioFileIdentifier`
- `private let metadataReader: AudioMetadataReader`
- `private let libraryIndex: MusicLibraryIndex`
- `private let config: AudioConfig` (from AppConfig)

**Init:**
- Accept all dependencies: `identifier`, `metadataReader`, `libraryIndex`, `config`

**canHandle(file: URL) -> Bool:**
- Delegate to `identifier.isRecognizedAudioFile(file)`
- Per user decision: non-audio files silently ignored (no log)

**process(file: URL) async throws -> TaskResult:**
Implement the full pipeline in order:

1. **Read metadata:**
   - `let metadata = try await metadataReader.read(from: file)`
   - On error: return `.error(description: "Failed to read metadata: \(error)")`

2. **Validate required fields (AUD-02):**
   - Check `metadata.hasRequiredFields(config.requiredMetadataFields)`
   - If fails: log at INFO level with `AppLogger.shared.info("Audio skipped: missing metadata", context: ["file": file.lastPathComponent, "missing": metadata.missingFields(config.requiredMetadataFields).joined(separator: ", ")])`
   - Return `.skipped(reason: "Missing metadata: \(missing fields)")`
   - Per user decision: file stays in ~/Downloads untouched

3. **Check duration (AUD-03):**
   - Compare `Int(metadata.durationSeconds)` against `config.minDurationSeconds` (integer comparison per research recommendation)
   - If `Int(metadata.durationSeconds) < config.minDurationSeconds`:
     - Log at INFO: "Audio skipped: duration too short"
     - Return `.skipped(reason: "Duration \(Int(metadata.durationSeconds))s < \(config.minDurationSeconds)s minimum")`

4. **Check duplicates (AUD-04 + AUD-05):**
   - Guard that title and artist are non-nil (already validated above if "title" and "artist" are in requiredMetadataFields, but handle defensively for when user disables those fields)
   - If title AND artist are available: `await libraryIndex.contains(title: title, artist: artist)`
   - If duplicate found:
     - Delete incoming file: `try FileManager.default.removeItem(at: file)` (AUD-05)
     - Log at INFO: "Audio duplicate deleted", context with file, title, artist
     - Return `.duplicate(title: title, artist: artist)`

5. **Move to ~/Music (AUD-06):**
   - Resolve destination: expand `config.destinationPath` (handle ~ expansion via `NSString(string:).expandingTildeInPath`)
   - Create destination directory if needed: `FileManager.default.createDirectory(at:withIntermediateDirectories:true)`
   - Let `destination = musicDir.appendingPathComponent(file.lastPathComponent)`
   - **Filename conflict handling (per user decision):**
     - If `FileManager.default.fileExists(atPath: destination.path)`:
       - This is NOT a metadata duplicate (already checked above) — it's a different song with same filename
       - Create `~/Music/possible_duplicate/` directory
       - Move to `possible_duplicate/` instead
       - Log at INFO: "Filename conflict — moved to possible_duplicate"
     - Else: `try FileManager.default.moveItem(at: file, to: destination)`
   - Update index: `await libraryIndex.add(title: title, artist: artist, url: finalDestination)`
   - Log at INFO: "Audio moved to ~/Music", context with file, title, artist
   - Return `.processed(action: "Moved to ~/Music")`

**Error handling:** Wrap FileManager operations (removeItem, moveItem) in do/catch. On permission errors (CocoaError.fileWriteNoPermission), log at ERROR level. On other errors, return `.error(description:)`.

**Important:** AudioTask must be Sendable. Since it holds an actor (MusicLibraryIndex) and Sendable structs (AudioFileIdentifier, AudioMetadataReader, AudioConfig), this should work. Mark as `struct AudioTask: FileTask, Sendable`.
