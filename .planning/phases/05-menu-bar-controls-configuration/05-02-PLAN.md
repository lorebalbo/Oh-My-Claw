  <action>
**Modify `OhMyClaw/App/AppCoordinator.swift`:**

1. **Add `pauseMonitoring()` public method:**

   This method stops new file detection but lets in-flight tasks finish:

   ```swift
   /// Pause monitoring — stop the FileWatcher but let in-flight tasks complete.
   /// The for-await event loop will exit naturally when the watcher's AsyncStream terminates.
   func pauseMonitoring() {
       appState.monitoringState = .paused
       fileWatcher?.stop()
       fileWatcher = nil
       // Do NOT cancel eventLoopTask — let in-flight iterations finish
       // The for-await loop will end when the watcher's AsyncStream finishes
       AppLogger.shared.info("Monitoring paused by user — in-flight tasks will complete")
   }
   ```

   Critical: Do NOT call `eventLoopTask?.cancel()`. The `for await fileURL in watcher.events` loop will exit naturally when `watcher.stop()` terminates the underlying `AsyncStream`. Any file currently being processed by `task.process(file:)` inside the loop body will complete normally.

2. **Add `resumeMonitoring()` public method:**

   ```swift
   /// Resume monitoring — restart the FileWatcher and rescan ~/Downloads.
   func resumeMonitoring() async {
       appState.monitoringState = .idle
       await startMonitoring()
       updateMonitoringState()
       AppLogger.shared.info("Monitoring resumed by user")
   }
   ```

   `startMonitoring()` already creates a new FileWatcher, starts FSEvents, scans existing files, and creates a new event loop task. Calling it here restarts the full pipeline.

3. **Refactor existing `stopMonitoring()` method:**

   The current `stopMonitoring()` cancels `eventLoopTask` — this is correct for a full app stop but wrong for pause. Refactor:

   - Keep `stopMonitoring()` as the full-stop method (used if needed for app termination)
   - Ensure `pauseMonitoring()` is used for user-initiated pause from the menu bar

   Update `stopMonitoring()` to also stop the icon animator:
   ```swift
   private func stopMonitoring() {
       healthPollingTask?.cancel()
       healthPollingTask = nil
       eventLoopTask?.cancel()
       eventLoopTask = nil
       fileWatcher?.stop()
       fileWatcher = nil
       iconAnimator.stopAnimating()
       AppLogger.shared.info("File monitoring stopped")
   }
   ```

4. **Remove or update `toggleMonitoring(_ isEnabled:)`:**

   This method was a bridge (before Plan 05-01's monitoringState was added). Replace it with logic that delegates to `pauseMonitoring()` / `resumeMonitoring()`:

   ```swift
   /// Called from the UI to toggle monitoring on/off.
   func toggleMonitoring(_ isEnabled: Bool) async {
       if isEnabled {
           await resumeMonitoring()
       } else {
           pauseMonitoring()
       }
   }
   ```

   This keeps backward compatibility with any existing call sites while using the correct pause semantics.

5. **Update `start()` method** if needed — ensure it checks the monitoring state properly. After Plan 05-01, it should already check `monitoringState != .paused`. Verify this is the case.

Key constraints:
- `pauseMonitoring()` MUST NOT cancel `eventLoopTask` — this is the critical semantic difference from `stopMonitoring()`
- `resumeMonitoring()` sets `.idle` first, then `startMonitoring()` creates the new event loop, then `updateMonitoringState()` derives actual state
- `stopMonitoring()` is private — only used internally for full shutdown
- `pauseMonitoring()` is public — used by UI
- `resumeMonitoring()` is async because `startMonitoring()` is async (it scans existing files)

```swift
@ViewBuilder
private var monitoringSection: some View {
    VStack(alignment: .leading, spacing: 8) {
        Text("Monitoring")
            .font(.caption)
            .foregroundStyle(.secondary)
            .textCase(.uppercase)

        Button(pauseResumeLabel) {
            Task {
                if case .paused = coordinator.appState.monitoringState {
                    await coordinator.resumeMonitoring()
                } else {
                    coordinator.pauseMonitoring()
                }
            }
        }

        // Inline ffmpeg warning (persistent)
        if !coordinator.appState.ffmpegAvailable {
            VStack(alignment: .leading, spacing: 4) {
                Label("ffmpeg not found", systemImage: "exclamationmark.triangle.fill")
                    .foregroundStyle(.orange)
                    .font(.caption)
                Text("Install via: brew install ffmpeg")
                    .font(.caption2)
                    .foregroundStyle(.secondary)
                Text("Audio files will be moved without conversion.")
                    .font(.caption2)
                    .foregroundStyle(.secondary)
            }
        }

        // Inline LM Studio warning (persistent until available)
        if !coordinator.appState.lmStudioAvailable {
            VStack(alignment: .leading, spacing: 4) {
                Label("LM Studio not reachable", systemImage: "exclamationmark.triangle.fill")
                    .foregroundStyle(.orange)
                    .font(.caption)
                Text("Ensure LM Studio is running with a model loaded.")
                    .font(.caption2)
                    .foregroundStyle(.secondary)
                Text("PDF classification is paused until available.")
                    .font(.caption2)
                    .foregroundStyle(.secondary)
            }
        }
    }
}

private var pauseResumeLabel: String {
    if case .paused = coordinator.appState.monitoringState {
        return "Resume Monitoring"
    }
    return "Pause Monitoring"
}

@ViewBuilder
private var appSection: some View {
    VStack(alignment: .leading, spacing: 8) {
        Text("App")
            .font(.caption)
            .foregroundStyle(.secondary)
            .textCase(.uppercase)

        Toggle("Launch at Login", isOn: launchAtLoginBinding)
            .toggleStyle(.switch)

        Button("Quit Oh My Claw") {
            NSApplication.shared.terminate(nil)
        }
    }
}

private var launchAtLoginBinding: Binding<Bool> {
    Binding(
        get: {
            SMAppService.mainApp.status == .enabled
        },
        set: { newValue in
            do {
                if newValue {
                    try SMAppService.mainApp.register()
                } else {
                    try SMAppService.mainApp.unregister()
                }
            } catch {
                AppLogger.shared.error("Launch at Login toggle failed",
                    context: ["action": newValue ? "register" : "unregister",
                              "error": error.localizedDescription])
            }
        }
    )
}
```

Key constraints:
- `SMAppService.mainApp.status` is read live in the getter — never cached
- `.register()` and `.unregister()` can throw — errors are logged, not propagated
- Available macOS 13+; our deployment target is macOS 14+
- The Pause button calls `coordinator.pauseMonitoring()` (synchronous — no await needed)
- The Resume button calls `await coordinator.resumeMonitoring()` (async — needs Task wrapper)
- Settings section is a placeholder — Plan 05-03 will replace it
- Width is 300px per user decision
